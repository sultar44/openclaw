## Gmail Webhook Finally Working (5:00 AM - 8:54 AM)

After several days of troubleshooting, the Gmail webhook pipeline is now fully operational.

### Problems Found & Fixed
1. **OAuth token expiry** — Moved gog OAuth app to Google Production status (permanent tokens, no more 7-day expiry)
2. **Transform syntax error** — `gmail-security.js` had nested backticks inside a template literal, crashing on every hook load. Fixed by escaping.
3. **Keyring access for launchd** — gog's file keyring backend needs `GOG_KEYRING_PASSWORD` env var. Added to `~/.openclaw/.env` (the official way to pass env vars to the gateway).
4. **Transform module not loading** — Gateway couldn't load ESM transform modules. Switched from custom transform to `messageTemplate` in the mapping config. This works.
5. **Model auth error** — Mapping had `model: anthropic/claude-sonnet-4-20250514` which wasn't allowed. Removed model override to use agent defaults. Also ran `openclaw gateway install --force` to sync tokens.

### Current Architecture
- Google Pub/Sub → Tailscale funnel (port 8788) → gog watch serve → POST to /hooks/gmail → messageTemplate mapping → agent session → delivers to #chloelogs (C0AELHCGW4F)
- Keyring: `file` backend with `GOG_KEYRING_PASSWORD` in `~/.openclaw/.env`
- No custom transform module (messageTemplate only)
- Security routing (trusted vs untrusted) is now in the messageTemplate text, not a JS transform

### 3-Day Trial
Ramon's rule: if it breaks again in 3 days, we abandon the webhook and switch to a simple cron job that polls email every 5-10 minutes. The cron fallback is ready to build if needed.

### Lessons Learned
- Don't restart the gateway from agent sessions — kills the connection
- The `.env` file is the correct way to pass env vars to the gateway (not plist editing)
- ESM transform modules don't work in OpenClaw hook mappings — use messageTemplate instead
- `openclaw gateway install --force` syncs config token to service token
- Always test with direct `curl` to the hook endpoint before testing the full pipeline
