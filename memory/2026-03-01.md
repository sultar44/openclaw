## Gmail Meltdown & Circuit Breaker (10 PM)

### What happened
- Gmail OAuth token expired (gog auth dead)
- Gmail Pub/Sub kept pushing notifications → each one spawned an LLM session that failed
- Runaway loop burned ~$30+ in wasted API calls and caused LLM timeouts across all sessions

### Root cause
- gog OAuth app is in Google "Testing" status → tokens expire in ≤7 days
- No rate limiting on gmail webhook → every push spawned a new session regardless of auth state
- The gmail-security.js transform existed but wasn't wired into the webhook mapping

### What we built
- **Circuit breaker** in `~/.openclaw/hooks/transforms/gmail-security.js`:
  - 5 hits in 60 seconds → trips and PERMANENTLY locks (no auto-reset cycling)
  - Sends ONE alert to #chloebot with re-auth instructions
  - Stays dead until manual reset (delete state file)
- Wired the transform into the webhook mapping (was using raw messageTemplate before, ignoring security routing entirely)
- Re-authed gmail OAuth (`gog auth add`)
- Gmail watcher confirmed running on port 8788

### Key learnings
- SMTP app password ≠ OAuth token (different auth mechanisms)
- OAuth tokens can't be stored/reused like passwords; require browser consent flow
- Need weekly auth health check in heartbeat (TODO: add to HEARTBEAT.md)
