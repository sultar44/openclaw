
## Gateway Restart Fix (Mar 10, 2026)
- **Problem:** Nightly auto-update cron ran *inside* the gateway process via `gateway.update.run` RPC. When it restarted the service, `launchctl bootout/bootstrap` could silently fail, leaving the gateway unloaded for hours. Happened Mar 9 and Mar 10.
- **Root cause:** Self-kill pattern. The gateway can't reliably restart itself because the process that issues the restart dies with it.
- **Fix implemented:**
  1. Disabled internal OpenClaw cron `6ac25d5e` ("Nightly OpenClaw Auto-Update")
  2. Created external LaunchAgent `ai.openclaw.auto-update` (runs `~/.openclaw/scripts/auto-update.sh` at 3:15 AM)
  3. Created external LaunchAgent `ai.openclaw.gateway-watchdog` (runs `~/.openclaw/scripts/gateway-watchdog.sh` every 5 minutes)
  4. Watchdog checks port 18789 responsiveness, kills zombies, handles launchd reload as last resort
  5. Auto-update script logs to ClickUp (same cron-id) and verifies gateway comes back after update
  6. Cleaned up 40MB error log from restart loop spam
- **Files created:**
  - `~/.openclaw/scripts/auto-update.sh`
  - `~/.openclaw/scripts/gateway-watchdog.sh`
  - `~/Library/LaunchAgents/ai.openclaw.auto-update.plist`
  - `~/Library/LaunchAgents/ai.openclaw.gateway-watchdog.plist`

## Cron Scheduler Issues (Mar 10 ~8:45-9:25 AM)
- **Problem:** After overnight gateway outage, 10 cron jobs were stuck in "running" state (zombie `runningAtMs` set but no actual process). Jobs had been "running" 50-80 min with normal runtimes of 0-228 seconds.
- **Stuck jobs:** Daily PPC, Product Master Sync, Cron Registry Sync, BSR Check, AWD Inventory, Sales & Traffic, Strategy Report, Rank Collection, Morning Self-Heal, Wholesale Pricing
- **Manual `cron run` is broken:** The CLI returns `enqueued: true` but jobs never actually execute. Tested multiple times across gateway restarts. This is an OpenClaw bug.
- **Workaround that works:** Gateway restart auto-fires missed jobs (schedule-based catchup). The disable→re-enable→`cron run` pattern does NOT work.
- **What we did:**
  1. Tried disable/enable + `cron run` — appeared to work (returned old cached state as "done") but jobs never ran
  2. Discovered stale gateway process (pid 36447) causing port conflicts on restart
  3. Hard killed orphan, did clean `launchctl bootstrap`
  4. Gateway restart auto-fired 5 missed jobs successfully (Amazon Data, SQP Retry, Self-Heal, Wholesale Pricing, Listing Monitor)
  5. The 8 originally disabled jobs got auto-re-enabled by gateway reload
- **Key lesson:** `openclaw cron run` (manual trigger) does not work. Only schedule-based auto-fire works. For manual reruns, may need to run underlying scripts directly.
- **Ramon preference:** When unsticking cron jobs, go one at a time ("slow and steady"), wait for confirmation before moving to next

## Gateway Outage & Recovery (7:53 AM - 3:00 PM)

### Root Cause
- Nightly auto-update cron (3:15 AM) upgraded OpenClaw v2026.3.7 → v2026.3.8
- Update sent SIGTERM to restart gateway, but the **LaunchAgent was not registered** with launchd
- Gateway stayed dead from 3:16 AM until Ramon opened Control UI at ~7:53 AM
- The plist file existed at ~/Library/LaunchAgents/ai.openclaw.gateway.plist but wasn't bootstrapped

### Dual-Process Conflict
- When Ramon opened Control UI, it started a **foreground** gateway process (PID 83687)
- Running `openclaw gateway install` registered the LaunchAgent, which started a **second** process (PID 93999)
- Two gateways fighting over port 18789 caused Slack agent turns to fail with "An unknown error occurred"
- Every message Ramon sent in #chloebot returned the error from ~1:05 PM to ~1:29 PM

### Resolution
- Killed the foreground process; LaunchAgent auto-restarted cleanly
- Gateway restart via SIGUSR1 at 1:29 PM cleared remaining bad state
- Confirmed zero agent errors after restart
- Slack test message delivered successfully

### Impact
- All morning cron jobs missed (3:15 AM - 7:53 AM): PPC, Wholesale Pricing, Morning Self-Heal, Cron Registry Sync, AWD Inventory, Daily Sales, Strategy Report, Daily Ranks
- Jobs resumed at their next scheduled times; none manually re-triggered (Ramon declined)

### Lesson
- If `openclaw gateway install` is run while a foreground gateway is already running, it creates a port conflict
- Always stop the existing process first, OR let the install handle it
- LaunchAgent can become unregistered during updates — this is the second time it's happened

### STP/PED Pricing Strategy Discussion

- **STP (Strikethrough Pricing)** is driven by Amazon's median price over a rolling window
- **CAN2P problem:** Liquidation sales ($8.99-$9.99) from Dec-Jan polluted the median. Current 90-day avg: $11.59, 180-day avg: $10.93. STP target is $14.99 strikethrough with $12.99 everyday.
- **CAN2P plan:** Hold $12.99, check back mid-April when liquidation data starts aging out. If still no STP, wait until late May.
- **CANSET works perfectly:** Clean price history at $26.99, spike works instantly
- **Coupon trick for low-velocity items:** Coupon-clipped sales register at the LIST price for STP calculations. So $19.99 with $5 coupon = $19.99 in Amazon's median. Good way to seed high data points on slow-movers.
- **90-day STP refresh cycle:** Every 90 days per SKU, spike price 25-35% above everyday for a few sales, drop back. For low-velocity items, hold the high price longer (24-48h vs 1-2h).
